
# Google Cloud Configuration Guide

This guide explains how to set up the necessary Google Cloud services (Gemini, Text-to-Speech, Drive, YouTube) and obtain the required credentials.

---

## 1. Create a Google Cloud Project

1. Go to the [Google Cloud Console](https://console.cloud.google.com/).
2. Click the specific project dropdown in the top bar.
3. Click **New Project**.
4. Name it `AI-Slop-Pipeline` (or similar) and click **Create**.
5. Select the newly created project.

---

## 2. Enable APIs

You need to enable the following APIs for your project:

1. Go to **APIs & Services > Library**.
2. Search for and **Enable** each of the following:
   - **Cloud Text-to-Speech API** (for voice generation)
   - **Google Drive API** (for downloading gameplay/uploading result)
   - **YouTube Data API v3** (for uploading shorts)
   - **Generative Language API** (if using Vertex AI, though Gemini often uses a separate MakerSuite key - see below)

---

## 3. Set up Google Cloud TTS (Service Account)

This is required for `GOOGLE_APPLICATION_CREDENTIALS`.

1. Go to **APIs & Services > Credentials**.
2. Click **+ CREATE CREDENTIALS** > **Service account**.
3. **Service account details**:
   - Name: `tts-service-account`
   - Click **Create and Continue**.
4. **Grant this service account access to project**:
   - Role: **Cloud Text-to-Speech API User** (or Editor if unsure).
   - Click **Continue** -> **Done**.
5. **Create Key**:
   - Click on the newly created service account email (e.g., `tts-service-account@...`).
   - Go to the **KEYS** tab.
   - Click **ADD KEY** > **Create new key**.
   - Select **JSON** and click **Create**.
   - A file will automatically download (e.g., `ai-slop-pipeline-12345.json`).
6. **Save Key**:
   - Rename this file to `service-account.json`.
   - Move it to a secure location on your machine (e.g., `C:\Users\YourName\.credentials\` or `~/.credentials/`).
   - **Update your `.env`**:
     ```bash
     GOOGLE_APPLICATION_CREDENTIALS=C:\Users\YourName\.credentials\service-account.json
     ```
     *(Use absolute path. On Linux/EC2 use `/home/ubuntu/.credentials/service-account.json`)*.

---

## 4. Set up Google Drive & YouTube (OAuth 2.0)

This is required for `GOOGLE_OAUTH_CREDENTIALS_PATH`.

1. Go to **APIs & Services > OAuth consent screen**.
   - Select **External** (unless you have a Google Workspace organization).
   - Click **Create**.
   - Fill in App Information (Name: AI Slop, Email: yours).
   - Add **Test Users**: Add your own Google email address (important for External apps).
   - Save and Continue.
2. Go to **APIs & Services > Credentials**.
3. Click **+ CREATE CREDENTIALS** > **OAuth client ID**.
   - Application type: **Desktop app**.
   - Name: `AI Slop Client`.
   - Click **Create**.
4. **Download JSON**:
   - Click the "Download JSON" icon related to the new ID.
   - Rename the file to `oauth-credentials.json`.
   - Move it to the same secure folder as above.
   - **Update your `.env`**:
     ```bash
     GOOGLE_OAUTH_CREDENTIALS_PATH=C:\Users\YourName\.credentials\oauth-credentials.json
     ```

---

## 5. Get Gemini API Key

This is required for `GEMINI_API_KEY`.

1. Go to [Google AI Studio (MakerSuite)](https://makersuite.google.com/app/apikey).
2. Click **Create API key**.
3. Select your Google Cloud project (created in step 1).
4. Copy the key string.
5. **Update your `.env`**:
   ```bash
   GEMINI_API_KEY=AIzaSy...
   ```

---

## Summary of Files You Should Have

| File | Source | .env Variable |
|------|--------|---------------|
| `service-account.json` | Cloud Console (Service Account Keys) | `GOOGLE_APPLICATION_CREDENTIALS` |
| `oauth-credentials.json` | Cloud Console (OAuth Client IDs) | `GOOGLE_OAUTH_CREDENTIALS_PATH` |
| `token.json` | Auto-generated by app on first run | `GOOGLE_OAUTH_TOKEN_PATH` |

**IMPORTANT**: 
- Never check these JSON files into Git.
- Add `*.json` to your `.gitignore` (already done).
